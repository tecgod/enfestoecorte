Sincronização com Google Sheets / Drive

O sistema funciona 100% offline por padrão, salvando tudo no seu navegador. A sincronização com o Google é opcional (uma cópia de segurança na nuvem, e também a forma de usar o sistema em mais de um aparelho) e só acontece quando você clicar nos botões de enviar/baixar abaixo. Nada é enviado para nenhum servidor além da sua própria conta Google.

Ainda não configurou? Veja o passo a passo completo

1. Criar o projeto do script

  1. Acesse script.google.com (entre com a mesma conta Google onde você quer salvar os dados).
  2. Clique em Novo projeto.
  3. Apague todo o conteúdo do arquivo Código.gs que abrir.
  4. Abra o arquivo Code.gs (fornecido junto com este sistema), copie todo o conteúdo e cole no editor do Apps Script.
  5. Dê um nome ao projeto, por exemplo "Controle Infesto e Corte" (canto superior esquerdo).
  6. Salve (ícone de disquete ou Ctrl+S).

2. Publicar como aplicativo da Web

  1. Clique em Implantar (canto superior direito) → Nova implantação.
  2. Em "Selecionar tipo", clique no ícone de engrenagem e escolha Aplicativo da Web.
  3. Configure: Executar como → Eu (seu e-mail); Quem pode acessar → Qualquer pessoa.
  4. Clique em Implantar.
  5. O Google vai pedir para você autorizar o script — é normal, é o script pedindo permissão para acessar Planilhas e Drive da sua própria conta. Clique em "Continuar" e depois em "Permitir".
  6. Copie a URL do aplicativo da Web que aparece (termina em /exec).

3. Conectar o sistema

  1. Cole a URL copiada no campo "URL do Google Apps Script (Web App)" acima e clique em Salvar.
  2. Clique em ⬆ Enviar tudo para o Google para fazer o primeiro envio.

Na primeira sincronização, o script cria automaticamente na sua conta uma planilha chamada "Controle Infesto e Corte - Dados" (com abas para Clientes, Cortes e Gastos) e uma pasta no Drive chamada "Controle Infesto e Corte - Anexos", onde ficam as fotos das etiquetas e das folhas de molde. Você pode acessar essa planilha a qualquer momento em drive.google.com.

Uso no dia a dia (mais de um aparelho)

  • ⬆ Enviar tudo para o Google: envia clientes, cortes e gastos salvos no navegador para a planilha, e as fotos ainda não enviadas para o Drive. Pode clicar quantas vezes quiser.
  • ⬇ Baixar do Google: traz os dados da planilha e substitui os dados salvos neste navegador/dispositivo. Útil para abrir o sistema em outro computador ou celular e continuar de onde parou.
  • Combine com a equipe: no fim do dia, quem estava usando clica em Enviar tudo para o Google; na manhã seguinte (ou em outro aparelho), a primeira coisa a fazer é Baixar do Google antes de lançar cortes novos, pra não trabalhar com dados desatualizados.
  • Não é uma sincronização automática em tempo real — é um envio/download manual, sob seu controle.

Perguntas comuns

"Enviar para o Google" deu erro de permissão.
Volte ao Passo 2 e confira se "Quem pode acessar" está como Qualquer pessoa. Se você editar o Code.gs depois de já ter implantado, é preciso ir em Implantar → Gerenciar implantações → editar (ícone de lápis) → Nova versão → Implantar para as mudanças valerem.

Posso usar minha própria planilha já existente?
O script cria e controla a planilha dele automaticamente para manter as abas no formato que o sistema espera. Não é recomendado apontar para uma planilha já existente com outro leiaute.

Dois cortes podem ficar com o mesmo código?
Não. O código é sempre data + número sequencial daquele dia. Se você importar um backup antigo que já tenha códigos, o sistema respeita os códigos existentes e só gera novos para cortes que ainda não têm um.

É seguro?
Sim — o script roda inteiramente dentro da sua própria conta Google. Nem a Anthropic nem terceiros têm acesso aos dados; eles ficam só na sua Planilha e no seu Drive.

🔐 Controle de acesso — liberar ou bloquear quem usa o sistema

O sistema exige login (usuário e senha) pelo menos uma vez por dia, em cada aparelho — e, com internet disponível, revalida essa sessão junto ao Google toda vez que o sistema é reaberto (veja mais abaixo). Quem pode entrar é definido por você, direto numa aba chamada "Usuários", criada automaticamente na mesma planilha do Google usada na sincronização (a mesma URL configurada acima), assim que a primeira tentativa de login acontecer. Cada pessoa também tem um papel: admin (acesso completo), usuario (acesso normal, sem a função "Google Sheets/Drive") ou leitura (só pode visualizar os dados já cadastrados — não consegue salvar, editar, excluir ou importar nada).

Como liberar uma pessoa

Abra a planilha "Controle Infesto e Corte - Dados" em drive.google.com, vá na aba Usuários e adicione uma linha com:

  • usuario — o nome de usuário que a pessoa vai digitar pra entrar.
  • senha — a senha dela.
  • status — escreva exatamente ativo.
  • papel — escreva admin para acesso completo (inclusive a função "Google Sheets/Drive"), usuario para acesso normal (sem essa função), ou leitura para acesso somente de visualização (a pessoa entra, vê cortes, clientes, relatórios e gastos já cadastrados, mas qualquer tentativa de salvar, editar, excluir ou importar dados é bloqueada com um aviso na tela).
  • limite — quantos aparelhos diferentes podem estar logados com esse mesmo usuário/senha ao mesmo tempo, no mesmo dia. Ex: 2 permite até 2 celulares/tablets usando esse login ao mesmo tempo; o 3º aparelho que tentar entrar é recusado até que um dos dois pare de usar. Deixe em branco (ou 0) para não ter limite nenhum. Se você reduzir esse número depois, os aparelhos que entraram por último (além do novo limite) perdem o acesso no próximo login/checagem deles.
  • validoAte — usada pra dar um período de teste a um cliente, com data de validade. Escreva uma data no formato dd/mm/aaaa (ex: 31/12/2026): a partir do dia seguinte a essa data, o acesso é recusado sozinho, sem você precisar fazer nada. Deixe em branco para "sem validade" (acesso não expira sozinho) — é o caso normal de um cliente já efetivado.
  • nome — nome da pessoa, só pra você identificar (opcional).

As colunas criadoEm e ultimoAcesso não precisam ser preenchidas por você — a coluna ultimoAcesso é atualizada sozinha pelo sistema a cada login bem-sucedido, pra você acompanhar quem andou usando e quando.

Dar um período de teste a um cliente (com expiração automática)

Preencha a coluna validoAte daquela linha com a última data em que o teste deve valer (ex: 15/08/2026). A partir do dia seguinte, o sistema recusa o acesso automaticamente daquele usuário, mostrando o aviso de período de teste expirado — sem você precisar lembrar de bloquear manualmente. Isso é checado tanto no login quanto na revalidação periódica (veja abaixo), então o corte acontece mesmo que a pessoa já estivesse com o sistema aberto no momento em que a validade vence.

Para efetivar o cliente (ele virou cliente de verdade, sem mais prazo de teste): simplesmente apague o conteúdo da célula validoAte daquela linha, deixando em branco. A partir daí, o acesso passa a valer normalmente, sem data de validade.

Derrubar um cliente à força (ex: atraso de pagamento)

Mude o status daquela linha de ativo para bloqueado (ou qualquer outra palavra). Não precisa apagar a linha nem mexer na validade ou na senha. O sistema confere isso com o Google toda vez que é aberto, e também a cada poucos minutos enquanto o sistema já está aberto (veja "Revalidação automática" logo abaixo) — então o corte de acesso acontece rapidamente, mesmo que a pessoa esteja usando o sistema naquele exato momento, sem precisar esperar ela fechar e abrir de novo.

O que o papel "leitura" pode e não pode fazer

Quem entra com papel leitura vê o sistema normalmente — telas, listas de cortes, clientes, relatórios e gastos já cadastrados aparecem do mesmo jeito. A diferença é só na hora de gravar algo: se a pessoa tentar salvar um corte novo, editar/excluir um corte, cliente ou gasto, ou cadastrar/remover um tecido ou nuance, ou importar um backup, o sistema mostra o aviso "Seu acesso é somente leitura" e não salva nada. Não é preciso mudar mais nada na planilha além do papel dessa pessoa.

Como funciona o limite de aparelhos

Cada aparelho (celular, tablet, computador) gera sozinho um identificador único na primeira vez que abre o sistema, e manda esse identificador junto com o login. O script guarda automaticamente, numa aba própria chamada "Sessoes_Dispositivos" (você não precisa mexer nela), quais aparelhos usaram cada login hoje, na ordem em que entraram. Um aparelho que já usou aquele login hoje continua podendo entrar de novo, desde que ainda esteja dentro do limite atual (contando pela ordem de chegada: quem entrou primeiro tem prioridade); um aparelho novo só entra se ainda não tiver batido no número definido em limite. No dia seguinte, a "vaga" libera sozinha — cada aparelho pede login de novo, do jeito que o sistema já funciona hoje.

Como bloquear uma pessoa

Na mesma aba, mude o status daquela linha de ativo para bloqueado (ou qualquer outra palavra — só ativo libera acesso). Não precisa apagar a linha nem mudar a senha. O sistema confere isso com o Google toda vez que é aberto e periodicamente enquanto já está aberto, então o bloqueio passa a valer rapidamente, mesmo que a pessoa já estivesse com o sistema aberto.

Revalidação automática da sessão

Toda vez que o sistema é aberto num aparelho que já tinha feito login antes, ele confirma com o Google, em segundo plano, se aquela sessão ainda é válida (status, validade do teste, papel atual, e limite de aparelhos simultâneos) — em vez de simplesmente confiar no que já estava salvo no navegador o dia inteiro. Além disso, essa mesma confirmação se repete sozinha a cada 10 minutos enquanto o sistema continuar aberto, mesmo sem a pessoa recarregar a página. Se algo mudou nesse meio-tempo (por exemplo, você bloqueou a pessoa, o teste dela expirou, ou o limite de aparelhos foi atingido em outro aparelho), o acesso é encerrado ali mesmo e a tela de login aparece de novo — é assim que um bloqueio por atraso de pagamento, por exemplo, "derruba" alguém mesmo que já estivesse usando o sistema.

Isso exige conexão com a internet: se a confirmação falhar por qualquer motivo (sem internet, problema na implantação do script etc.), o sistema não deixa passar — encerra a sessão e pede login de novo, mostrando o motivo do erro na tela.

Para você conferir com quem está logado em cada aparelho, o nome do usuário aparece no topo da tela, ao lado do status de sincronização (ex: "👤 a6").

A aba "Usuários" é criada apenas uma vez e o sistema nunca apaga ou reescreve as linhas dela sozinho — diferente das outras abas (Clientes, Cortes, Gastos), que são atualizadas a cada "Enviar para o Google". Você tem controle total e manual sobre quem está na lista.

📖 Leia-me — como o sistema calcula

Este guia explica, campo por campo, todas as contas que o sistema faz sozinho no formulário de corte. Consulte sempre que precisar conferir ou lembrar de uma fórmula.

Tipo de média: Ímpar × Pasano

Essa opção não muda a quantidade de peças — ela só muda como a média de consumo por peça é calculada, porque afeta o jeito que o tecido é enfestado (esticado em camadas) antes de cortar:

  • Ímpar: média (m) = Comprimento do enfesto ÷ Qtd. peças riscadas.
  • Pasano: média (m) = (2 × Comprimento do enfesto) ÷ Qtd. peças riscadas — o tecido é dobrado ao meio antes de cortar, então o comprimento "rende o dobro" nessa conta.

Total de peças

Total de peças = Qtd. enfesto × Qtd. peças riscadas.

Essa fórmula é a mesma para Ímpar e para Pasano — o tipo de média não divide nem altera o total de peças, só entra na conta da média (acima) e do consumo (abaixo).

Consumo total de tecido

Consumo total (m) = Total de peças × Média (m).

Valor total do corte

Valor total = Total de peças × Valor (informado no campo "Valor").

Tempo gasto e valor por hora

Cada corte pode ter uma ou mais sessões de trabalho (data + início + fim). Dentro de uma mesma sessão, se o horário de fim for antes do de início, o sistema entende que passou da meia-noite e soma 24h. O Tempo gasto do corte é a soma da duração de todas as sessões preenchidas — o tempo entre uma sessão e outra (folga, domingo, virou o dia sem trabalhar) nunca entra na conta. Assim, um corte que começou no sábado à tarde, ficou parado no domingo e terminou na segunda deve ter duas sessões: uma no sábado e outra na segunda; só as horas realmente trabalhadas em cada uma são somadas. Valor por hora = Valor total ÷ (Tempo gasto em horas).

Molde em grade — quantidade de peças por tamanho

Quando o molde é do tipo Grade (vários tamanhos no mesmo corte), cada linha de tamanho tem sua própria "Qtd. desenho" (quantas vezes aquele tamanho se repete no risco/desenho do molde). A quantidade de peças daquele tamanho é:

Qtd. de peças do tamanho = Qtd. desenho (daquele tamanho) × Qtd. enfesto (campo geral do corte).

Ou seja, a mesma regra do "Total de peças" acima, aplicada tamanho por tamanho. A soma das linhas da grade é exibida como o total de tamanhos, logo abaixo da lista.

Código do corte

Gerado automaticamente: data do corte + número sequencial daquele dia (ex: primeiro corte lançado no dia vira "...-01", o segundo "...-02", e assim por diante).

❓ Manual de ajuda / Tutorial

Guia completo de como usar o sistema, do zero. Clique em cada tópico abaixo para abrir. Se a dúvida for sobre alguma fórmula ou conta específica, veja também o 📖 Leia-me (como o sistema calcula), no mesmo menu de configurações.

1. Entrando no sistema (login)

Ao abrir o sistema, é preciso entrar com usuário e senha cadastrados pelo administrador. Depois de entrar uma vez, o acesso fica liberado naquele aparelho até o fim do dia — no dia seguinte, pede login de novo. Isso exige internet no momento de entrar (mesmo que o resto do sistema funcione offline depois).

A função "☁ Google Sheets/Drive", no menu de configurações, aparece somente para usuários com perfil de administrador — os demais usuários não veem essa opção.

Esqueceu a senha ou não consegue entrar? Fale com o administrador do sistema — só ele pode cadastrar, liberar ou bloquear usuários.

2. Visão geral do sistema

O sistema tem duas abas principais, no topo da tela:

  • 📐 Infesto e Corte: onde você cadastra cada corte realizado (formulário) e consulta o histórico de tudo que já foi lançado.
  • 📊 Relatórios e Indicadores: painéis com totais, gráficos, rankings e o controle de gastos, pra acompanhar o financeiro e a produtividade.

No canto superior direito fica o ícone de engrenagem (⚙), com o menu de configurações: exportar CSV, importar/salvar backup, sincronizar com Google, este manual, o "Leia-me" com as fórmulas e a opção de sair (encerrar sessão).

O sistema funciona 100% offline por padrão — tudo é salvo automaticamente no navegador do próprio aparelho, sem precisar de internet nem de clicar em "salvar" o tempo todo (exceto o formulário de corte em si, que tem um botão "Salvar corte"). A sincronização com Google é opcional.

3. Antes de lançar o primeiro corte: cadastre cliente, tecido e nuance

Esses três cadastros aparecem como listas (selects) no formulário de corte. Você pode criá-los na hora, sem sair do formulário:

Cliente

  • No campo "Cliente", clique no botão + ao lado da lista pra cadastrar um novo (nome da empresa, responsável, WhatsApp e observações — só o nome é obrigatório).
  • Clique no botão 📋 pra abrir "Clientes cadastrados", onde dá pra editar ou excluir clientes já existentes.

Tecido e Nuance

  • Mesma lógica: botão + ao lado de cada campo abre uma janela pra cadastrar, editar ou excluir os itens da lista (tecidos como "Viscose", "Malha PV"; nuances como "Preto", "Off White").

Não é obrigatório cadastrar tudo antes — você pode ir cadastrando aos poucos, conforme for lançando os cortes.

4. Lançando um corte — passo a passo

Na aba 📐 Infesto e Corte, o formulário "Novo corte" fica no topo. Preenchendo os campos, o sistema já calcula tudo sozinho (total de peças, consumo, valor total etc — veja as fórmulas no "Leia-me"). O código do corte é gerado automaticamente assim que a data é preenchida.

Campos principais

  • Cliente, Data, Peça, Referência, Modelo, Tecido, Nuance: identificação do corte.
  • Molde — Único ou Grade: escolha Único quando o corte tem um só tamanho; escolha Grade quando o mesmo corte tem vários tamanhos (P, M, G...) — nesse caso, uma caixa extra aparece pra você lançar a quantidade de cada tamanho (veja o tópico 4).
  • Qtd. enfesto e QTD. peças riscadas: o "Total de peças" é calculado sozinho a partir desses dois campos.
  • Comprimento do enfesto (m) e o tipo de Média (Ímpar ou Pasano): definem a média de consumo por peça e o consumo total de tecido — a diferença entre Ímpar e Pasano está explicada no "Leia-me".
  • Valor/pç: valor pago por peça; o "Total (R$)" a receber é calculado sozinho.

Sessões de trabalho (início/fim)

Mais abaixo no formulário, a caixa "Sessões de trabalho" registra quanto tempo você levou pra fazer o corte. Clique em + Sessão pra adicionar uma linha: primeiro informe a Data, depois o horário de Início, o de Fim, e a Duração daquela sessão aparece calculada automaticamente. Se o corte foi feito em dias separados (por exemplo, começou no sábado e terminou na segunda), adicione uma sessão para cada dia — o tempo entre as sessões não é contado. O ✕ ao lado da data remove aquela sessão inteira.

Fotos

  • Etiquetas: pode enviar uma ou várias fotos (ex: etiquetas do tecido, do rolo).
  • Folha do molde: uma foto da folha de risco/molde enviada pela empresa.

No celular, esses botões abrem direto a câmera ou a galeria, dependendo do que você escolher.

Salvando

Clique em Salvar corte para gravar. O botão Limpar zera o formulário sem salvar — use antes de começar um novo corte do zero, ou depois de salvar, se preferir.

5. Molde em Grade — lançando vários tamanhos no mesmo corte

Ao escolher Grade no campo "Molde", aparece a caixa "Quantidade por tamanho da grade". Clique em + Tamanho pra cada linha (P, M, G, GG...) e informe a Qtd. desenho de cada tamanho — ou seja, quantas vezes aquele tamanho se repete no risco/desenho do molde. O sistema calcula sozinho a quantidade de peças de cada tamanho e mostra o total somado no rodapé da caixa ("Total por tamanho"), junto com um aviso se esse total não bater com o "Total de peças" do corte.

6. Histórico de cortes — buscar, filtrar, editar, imprimir

Logo abaixo do formulário, na aba 📐 Infesto e Corte, fica a tabela "Histórico de cortes" com tudo que já foi salvo.

  • Busca e filtros: use o campo de texto (busca por código, modelo, referência ou peça) e os filtros de cliente, tipo de média e período (data inicial/final) para encontrar cortes específicos.
  • Editar um corte: clique na linha (ou no ícone de edição, se houver) para abrir aquele corte de volta no formulário, com todos os dados preenchidos, prontos para alterar e salvar novamente.
  • Selecionar vários: marque as caixinhas à esquerda de cada linha (ou a caixinha do cabeçalho, pra marcar todos os filtrados de uma vez). Com um ou mais selecionados, aparece uma barra com o botão 🖨 Imprimir selecionados, que gera um resumo pronto para impressão de todos os cortes marcados, e o botão Limpar seleção.
7. Aba Relatórios e Indicadores

Reúne o financeiro e a produtividade da confecção. No topo, escolha o período de análise (Hoje, Esta semana, Este mês, Tudo, ou um período personalizado com as duas datas) — todos os painéis abaixo se ajustam automaticamente a esse período.

  • Visão financeira: receita, gastos, lucro líquido, margem de lucro e ticket médio por corte.
  • Força de trabalho e produtividade: tempo total trabalhado (soma das sessões de todos os cortes do período), tempo médio por corte, valor médio da hora, peças por hora e valor médio por peça.
  • Mão de obra: custo × faturamento: pega só os gastos lançados na categoria "Mão de obra" no período e calcula quanto isso representa por peça e por hora (mesma base do painel de produtividade), comparando com o "Valor médio por peça"/"Valor médio da hora" que você cobra. Útil pra quem trabalha majoritariamente com corte por encomenda (tecido do cliente, custo de material mínimo) e quer saber se o valor cobrado está cobrindo bem a mão de obra — a margem fica vermelha se o custo de mão de obra superar o valor cobrado.
  • Evolução no período: gráfico comparando receita × gastos ao longo do tempo.
  • Desempenho por tipo de peça: ranking de quais peças (Vestido, Calça...) rendem mais R$/hora — só entram nessa conta os cortes com sessões de início/fim preenchidas.
  • Top clientes por faturamento e Gastos por categoria: rankings pra identificar seus melhores clientes e onde o dinheiro está sendo gasto.
  • Resumo rápido: uma caixa recolhível no fim da página, com totais simples (cortes, peças, metros, total a receber) e seus próprios atalhos de período, independente do filtro de cima.
8. Controle de gastos

Na aba 📊 Relatórios e Indicadores, no painel "Gastos registrados", clique em + Novo gasto para lançar uma despesa: data, categoria (Tecido/Insumos, Mão de obra, Aluguel/Estrutura, Manutenção/Equipamento, Transporte, Impostos/Taxas, entre outras), descrição e valor. Os gastos lançados entram automaticamente na "Visão financeira" (lucro líquido, margem) e no painel "Gastos por categoria", sempre respeitando o período selecionado no topo da página.

9. Backup, exportação e sincronização com Google

Tudo fica salvo automaticamente no navegador do aparelho, mas vale ter backups — principalmente antes de trocar de celular/computador ou limpar os dados do navegador. As opções ficam no menu de configurações (ícone ⚙, canto superior direito):

  • 📄 Exportar CSV: baixa uma planilha simples (CSV) com os cortes lançados, pra abrir no Excel/Google Sheets.
  • 💾 Salvar backup: baixa um arquivo com todos os dados do sistema (clientes, cortes, gastos, fotos) — o jeito mais completo de guardar uma cópia de segurança.
  • 📥 Importar backup: carrega de volta um arquivo salvo anteriormente com "Salvar backup".
  • ☁ Google Sheets/Drive: sincronização opcional com a sua própria conta Google, útil para usar o sistema em mais de um aparelho ou ter uma cópia na nuvem. Tem um passo a passo completo dentro dessa própria janela, incluindo perguntas frequentes.
10. Dicas rápidas de uso no celular
  • No teclado do celular, ao preencher os campos, use a tecla Enter/Próximo — o sistema pula automaticamente para o campo seguinte em vários pontos do formulário (data, sessões de trabalho, horários), poupando toques.
  • Os campos de horário (Início/Fim das sessões) aceitam só números — digite os 4 dígitos (ex: "0800" vira "08:00") que o sistema formata sozinho.
  • Os botões de foto (Etiquetas, Folha do molde) abrem a câmera do celular diretamente.
  • Toda tabela e formulário são roláveis — se algo parecer cortado na tela, deslize para ver mais.
